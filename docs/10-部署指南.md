# 10 — 部署指南

## 本地部署（开发调试）

### 后端

```bash
cd backend

# 虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows

# 安装
pip install -e .

# 配置
cp .env.example .env
# 编辑 .env 填入 LLM Key 和数据源

# 启动
python run.py
# 监听 0.0.0.0:8000
```

### 前端

```bash
cd frontend

# 安装
npm install

# 开发模式（热重载）
npm run dev
# 监听 0.0.0.0:5180

# 生产构建
npm run build
# 输出到 dist/
```

### 开发模式特点

| 特性 | 说明 |
|------|------|
| 热重载 | 前端修改自动刷新 |
| Vite Proxy | 前端 `/v1/*` 自动代理到后端 `:8000` |
| Debug 日志 | 后端 uvicorn 默认 DEBUG 级别 |
| SQLite 文件 | 数据存储在 `backend/data/chatsql.db` |

## Docker 部署（生产推荐）

### docker-compose.yml

```yaml
version: "3.8"

services:
  backend:
    build:
      context: .
      dockerfile: Dockerfile
      target: backend
    ports:
      - "8000:8000"
    volumes:
      - chatsql-data:/app/data
      - ./backend/.env:/app/.env:ro
    environment:
      - CHATSQL_LLM_PROVIDER=${CHATSQL_LLM_PROVIDER:-openai}
      - CHATSQL_LLM_API_KEY=${CHATSQL_LLM_API_KEY}
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8000/healthz"]
      interval: 30s
      timeout: 5s
      retries: 3

  frontend:
    build:
      context: .
      dockerfile: Dockerfile
      target: frontend
    ports:
      - "5180:80"
    depends_on:
      - backend
    restart: unless-stopped

volumes:
  chatsql-data:
```

### Dockerfile

```dockerfile
# ---- Backend ----
FROM python:3.12-slim AS backend
WORKDIR /app
COPY backend/pyproject.toml .
RUN pip install --no-cache-dir .
COPY backend/ .
RUN mkdir -p data
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

# ---- Frontend ----
FROM node:20-alpine AS frontend-build
WORKDIR /app
COPY frontend/package*.json ./
RUN npm ci
COPY frontend/ .
RUN npm run build

FROM nginx:alpine AS frontend
COPY --from=frontend-build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### 一键启动

```bash
# 克隆
git clone https://github.com/lushui1/chatsql.git
cd chatsql

# 配置
cp backend/.env.example backend/.env
vim backend/.env  # 填入 LLM Key

# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down
```

### Nginx 配置（nginx.conf）

```nginx
server {
    listen 80;
    server_name _;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /v1/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;  # SSE 流式需要关闭缓冲
        proxy_cache off;
    }

    # 健康检查代理
    location /healthz {
        proxy_pass http://backend:8000;
    }
}
```

## 生产环境注意事项

1. **HTTPS**：生产环境务必配置 SSL 证书
2. **CORS**：限制 `allow_origins` 为实际域名
3. **API Key**：设置 `CHATSQL_ADMIN_KEY` 保护管理接口
4. **日志**：配置日志文件轮转，避免磁盘占满
5. **备份**：定期备份 `data/` 目录
6. **内存**：跨库查询大数据量时注意内存限制
7. **超时**：LLM 调用设置合理超时（默认 120 秒）

## 常见部署问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端 502 | 后端未启动 | 先启动后端 |
| SSE 不流式 | Nginx 缓冲 | `proxy_buffering off` |
| 数据源连不上 | 驱动未安装 | `pip install aiomysql` 等 |
| OOM Killed | 内存不足 | 减小查询范围或增加内存 |
| 配置丢失 | 未持久化 | 检查 `data/` 目录是否挂载 volume |
