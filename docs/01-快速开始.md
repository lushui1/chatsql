# 01 — 快速开始

## 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.11+ | 后端运行时 |
| Node.js | 18+ | 前端构建 |
| pip | 23+ | Python 包管理 |
| npm | 9+ | Node 包管理 |
| Docker（可选） | 24+ | 容器化部署 |

## 方式一：本地部署（开发调试）

### 1. 克隆仓库

```bash
git clone https://github.com/lushui1/chatsql.git
cd chatsql
```

### 2. 启动后端

```bash
cd backend

# 创建虚拟环境
python -m venv .venv
# Windows
.venv\Scripts\activate
# Linux/Mac
source .venv/bin/activate

# 安装依赖
pip install -e .

# 复制环境变量
cp .env.example .env
# 编辑 .env，填入你的 LLM API Key

# 启动
python run.py
# 或
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

后端启动在 `http://127.0.0.1:8000`

### 3. 启动前端

```bash
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

前端启动在 `http://localhost:5180`

### 4. 验证

```bash
# 检查后端健康
curl http://127.0.0.1:8000/healthz
# {"ok":true,"service":"chatsql"}

# 检查前端代理
curl http://localhost:5180/healthz
# 同上说明 proxy 正常
```

## 方式二：Docker 部署（生产推荐）

### 1. 克隆仓库

```bash
git clone https://github.com/lushui1/chatsql.git
cd chatsql
```

### 2. 配置环境变量

```bash
cp backend/.env.example backend/.env
# 编辑 backend/.env，填入 LLM API Key 和数据源配置
```

### 3. 一键启动

```bash
docker-compose up -d
```

启动后：
- 前端：`http://localhost:5180`
- 后端 API：`http://localhost:8000`
- 健康检查：`http://localhost:8000/healthz`

### 4. 查看日志

```bash
# 所有服务
docker-compose logs -f

# 仅后端
docker-compose logs -f backend

# 仅前端
docker-compose logs -f frontend
```

### 5. 停止

```bash
docker-compose down
```

## 首次配置

启动后打开 `http://localhost:5180`，点击左下角 ⚙️ 设置：

### 配置 LLM

1. 切换到 🤖 大模型 Tab
2. 选择 Provider（如 openai）
3. 填入 API Key
4. 选择模型（如 gpt-4o-mini）
5. 点击 💾 保存

### 配置数据源

1. 切换到 🗄️ 数据源 Tab
2. 点击 ＋ 添加数据源
3. 填入连接信息（名称、类型、主机、端口、库名、用户名、密码）
4. 点击 🔗 测试连接
5. 测试通过后点击 💾 保存

## 第一次对话

1. 点击左侧 ＋ 新建对话
2. 输入问题，例如：
   - "帮我查一下最近7天的订单数量"
   - "按分拣中心统计今天的包裹量"
   - "哪个网点的签收率最高？"
3. Agent 会自动分析、生成 SQL、执行并返回结果和图表

## 内置示例数据

ChatSQL 内置了 DuckDB 示例数据，无需配置外部数据库即可体验：

| 表名 | 说明 | 示例数据 |
|------|------|---------|
| orders | 物流订单 | 3 条示例记录 |
| routes | 运输路线 | 路线信息 |
| sort_center | 分拣中心 | 中心编码和名称 |

首次启动时自动加载，可以直接问：
- "查一下所有订单"
- "按分拣中心统计订单数"
